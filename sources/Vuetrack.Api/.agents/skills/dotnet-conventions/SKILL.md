# .NET conventions

Conventions for a layered .NET/C# codebase. Each topic lives in its own
reference file — read the one that matches the task.

## Conventions

- **File & type naming** — which suffix a class and its file get based on the
  role it plays (`Contract`, `Filter`, `Model`, `Container`, `Request`,
  `Response`). Read [references/file-naming.md](references/file-naming.md) when
  creating, renaming, moving, or reviewing a data-carrying type.
